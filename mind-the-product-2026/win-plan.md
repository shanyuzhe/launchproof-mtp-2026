# Win Plan

## Strategic Read

This is a product-minded hackathon with a hard Novus.ai requirement, so polish, story, and measurable shipped behavior matter as much as implementation. A technically complex tool that is hard to understand will lose to a narrower tool that makes the judge say: "I would use this before my next launch."

The project should feel like a finished workflow, not a collection of AI outputs.

## Target User

Primary:

Solo builder or PM using AI coding tools to create a product quickly.

Pain:

They can generate software faster than they can validate whether it solves a real problem, works through the core path, and can be explained clearly.

Desired outcome:

Ship with a crisp product narrative, a tested primary flow, and a short evidence packet.

## Scoring Bet

Judging is equally weighted across four categories:

- Product Thinking: lead with the AI-builder launch-readiness pain.
- Craft and Execution: make one end-to-end workflow feel polished.
- Originality and Ambition: turn product analytics into a launch copilot, not another idea generator.
- Shippedness: deploy publicly, install Novus, generate behavior data, and show dashboard evidence.

## Build Scope

Day 1:

- create landing-free app shell with the main workflow as first screen
- project intake form
- launch brief generator
- flow checklist generator
- Novus installation path decided and stubbed

Day 2:

- risk register
- evidence board
- exportable Devpost draft
- sample project seed
- meaningful event names mapped for Novus

Day 3:

- browser validation or product-behavior evidence report
- demo script
- visual polish
- README and hosted demo
- Novus dashboard screenshot generated

Final polish:

- record 2 to 3 minute video
- write Devpost submission
- make screenshots
- test the live app on desktop and mobile
- verify Novus still receives events after deploy

## Demo Story

1. Open with the shift: AI made building cheap, but launch judgment is still scarce.
2. Show a messy idea: "AI meeting follow-up assistant for remote teams."
3. Run it through LaunchProof.
4. Get a launch brief, critical flows, risks, and evidence board.
5. Trigger tracked actions and show how Novus proves usage behavior.
6. Show how this becomes a better demo, better QA checklist, and better launch decision.
7. Close with: LaunchProof is the product layer for the AI-builder era.

## Winning Differentiators

- Not just another chat app.
- Serves the hackathon community itself.
- Combines product thinking, QA, analytics, and storytelling.
- Produces submission assets judges can immediately understand.
- Can become open source and useful after the event.

## Risks

- Too much generated text: keep outputs structured and scannable.
- Looks like a template generator: add an evidence/verification loop.
- Too broad: focus on launch readiness, not full product management.
- No real demo: include a seeded sample and a live end-to-end workflow.
- Novus not working: install early and test with real interactions before building polish.
