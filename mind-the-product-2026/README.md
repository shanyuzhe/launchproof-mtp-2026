# Mind the Product 2026 Hackathon

## Working Project

LaunchProof

An AI-assisted launch readiness workspace for solo builders, PMs, and small teams. It turns a rough product idea or demo URL into a concrete shipping packet:

- target user and pain statement
- core user flows to prove
- acceptance checks for the demo
- launch risks and mitigations
- Novus-backed evidence board for the final pitch
- Devpost-ready story and demo script

## Why This Fits

The rules reward Product Thinking, Craft and Execution, Originality and Ambition, and Shippedness equally. LaunchProof fits because it helps AI builders move from prototype to product evidence.

The Novus.ai requirement is not a checkbox for us. LaunchProof should use product behavior evidence as the center of the workflow: what did users try, where did they get value, and what should the builder fix before launch?

## Competition Positioning

One-liner:

AI makes everyone faster at building. LaunchProof helps everyone prove what is ready to ship.

Judge-facing promise:

LaunchProof helps a team go from messy idea to launch-ready evidence in under 20 minutes, with clear flows, risks, Novus-measurable events, and a stronger demo.

## MVP

The first useful version should do five things:

1. Capture a product idea, user, problem, and optional live URL.
2. Generate a concise launch brief.
3. Produce 3 to 5 critical user flows and acceptance checks.
4. Build a risk register with mitigation steps.
5. Track meaningful actions through Novus.ai.
6. Export a polished Devpost/pitch packet.

Stretch:

1. Run browser checks against a live URL.
2. Generate a demo script with timestamps.
3. Save multiple projects locally.
4. Add a public sample gallery.
5. Show a lightweight in-app analytics summary that mirrors the Novus story.
